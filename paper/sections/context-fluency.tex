\section{Context Fluency: An Emerging Skill}

Our case study and the broader practitioner literature suggest that effective
agentic coding requires a skill set distinct from both traditional programming
and prompt engineering. We propose the term \emph{context fluency} to describe
this capability: the ability to create rich, structured context that AI agents
can act on effectively. Context fluency encompasses capturing domain expertise,
value judgments, and design intent in machine-legible form---not as one-shot
prompts, but as persistent informational environments within which agents
operate. Prompt engineering optimizes individual instructions; context fluency
is \emph{upstream} of prompting, concerned with the entire informational
architecture surrounding agent execution~\cite{anthropic2025context,
gartner2025context}.

Where the methodology (Section~\ref{sec:methodology}) describes a
process of sequential phases producing defined artifacts, context
fluency describes the practitioner skill that makes the process
effective. The distinction parallels that between a recipe and culinary
judgment: the recipe structures the work, but the outcome depends on
the cook's ability to execute it.

We identify four components of context fluency:
\begin{description}
  \item[Decomposition:] Breaking problems into discrete, parallelizable
    tasks with clear boundaries, enabling concurrent agent execution.
  \item[Specification:] Describing not just \emph{what} to build but
    \emph{why}, so agents can make aligned micro-decisions without human
    intervention.
  \item[Constraint definition:] Knowing what to exclude, simplify, or
    defer---scope management as a first-class concern.
  \item[Domain encoding:] Externalizing tacit
    knowledge~\cite{polanyi1966tacit} that agents cannot generate on
    their own---directly instantiating what Nonaka and
    Takeuchi~\cite{nonaka1995knowledge} term \emph{externalization},
    converting tacit knowledge into explicit, communicable form.
    Twenty minutes of dictated pedagogical intuitions
    produced the context that made the Socratic tutor correct on
    first implementation.
\end{description}

These components map to established frameworks. Backward
design~\cite{wiggins1998understanding} contributes outcome-driven preparation.
Polanyi's tacit knowledge~\cite{polanyi1958personal} frames the externalization
challenge: practitioners ``know more than they can tell,'' and context fluency
is the discipline of telling it anyway. We also observe a connection to pedagogical scaffolding: context fluency
resembles lesson planning, where the teacher structures an environment so the
learner (or agent) can do meaningful work independently. Both disciplines are
fundamentally about enabling others to act within a defined informational
context.

If context fluency is a distinct and identifiable skill, the implications extend
beyond individual practice. For developer education, it suggests that curricula
should cultivate specification, decomposition, and domain-encoding abilities
alongside traditional programming competencies. For hiring, it implies that
practitioners with strong domain knowledge and pedagogical instincts---the
ability to structure information for consumption by others---may be
disproportionately effective in agentic workflows~\cite{dakhel2025humanai}.
For tool design, it reframes preparation aids such as structured task
systems~\cite{yegge2025beads, emanuel2025beadsrust}, specification
frameworks~\cite{github2025speckit}, and context engineering
platforms~\cite{anthropic2025context} as essential infrastructure rather than
optional enhancements.
